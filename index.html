    Estructura semantica del HTML

    • header: Representa la cabecera de la pagina o de una seccion, contiene elementos como logotipos, menus de navegacion y titulos
    • nav: Elemento de navegacion, mayormente contiene enlaces que funcionan como un guia dentro de la página.
    • main: Cuerpo principal de la página, es único.
    • footer: Es el final de la pagina y contiene elementos de bisuqeda, enlaces relacionados e información del autor.
    • article: Elemento flotante, agrupa el contenido relacionado dentro de pequeños elementos que conforman una sección.
    • section: Sección generica de un documento, suele estar acompañado de un encabezado.
    • aside: Contenido lateral como un menu lateral o un bloque de anuncios
    • figure: Elementos auto contenidos como las imagenes o gráficos.
    • figcaption:Añade una etiqueta a un elemento/
    • div: Elementos semanticos genericos que se usan para dividir, se pueden anidar sin restricciones.
    1. Hn: Los encabezados comienzan por el 1 y pueden haber los que se requieran
    2. Listas: Elementos numerados o no, pero
    3. Arial labels:Mejora la accesibilad del contenido para debiles visuales.

    Ayudan a los motores de busqueda a ordenar el contenido en los motores de busqueda ya que dan contenido adicional, si bien los elementos semanticos no afectan el diseño de una pagina, son fundamentales para la estructura.

    Formularios

    Un formulario en HTML es una herramienta que permite a los usuarios enviar datos a un servidor

    • Elementos del formulario

      1. form: Contenedor principal de los elementos de formulario
        Formulario principal
      2. input Entrada de texto generica, el tipo de texto se define por type
      3. label Etiqueta que describe el proposito del campo en un popup
      4. txtarea Un input pero que acepta muchas mas lineas
      5. button Botones interactivos

    • Campos de texto

      • Se utiliza la etiqueta input con una variación de los modificadores del tipi Type

      1. text: Campos genericos
      2. email: El navegador valida que el texto ingresado tiene el formato de un correo.
      3. password: Oculta la entrada de texto con *

    • Boton de envio

      Modificador de la etiqueta submit de button e input respectivamente

      1. button:Envía los datos del formulario de un boton
      2. inputTambien envia los datos pero no permite contenido adicional Este input no necesita cerrar etiqueta.

    • Etiquetas de agrupación

      1. fieldset: Agrupa elementos dentro de un formulario
      2. legend: Proporciona una descripcion a un campo dentro del fieldset

    Sesion 7: Trabajo en Git

    La pagina web se puede deployar en el propio github con el llamado github pages

    Archivo README.md

    Este archivo con extensión markDown contiene la descripción del proyecto y es de vital importancia para el desarrllo de software y debe contener lo siguiete:

    • Proposito del proyecto: Explica brevemente el proyeto.
    • Instrucciones de ejecucion: Como se usa, instala y corre. Además de los requisitos
    • Información del autor: Indica quien desarrolló el proyecto
    • Licencia: En caso de ser necesario se debe mencionar si corre bajo cierta licencia.

    GitHub Pages

    Herramienta gratuita que ofrece GitHub para alojar sitios web estáticos directamente desde tu repositorio.

    A continuacion se describen los pasos necesarios para deplyar la pagina web de forma estática.

    1. Configuracion de GitHub: Desde la página se debe acceder a las configuracions del repo.
    2. Activar GitHub Pages: Desde los ajustes se accede a la pestaña Pages y se debe seleccionar la rama del repositorio que contiene el elemento index.html
    3. Guardar y archivar: Al guardar github generará una liga a la que se podrá acceder.
    4. Enlace: La generacion del link puede tardar varios minutos.
